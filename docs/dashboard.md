# Dashboard Screenshot Placeholder

This repository intentionally does **not** store binary screenshots.

If you want to generate a local dashboard screenshot for documentation:

1. Start the stack locally (`make up`, then `make dev`).
2. Open the web UI in a browser (default: `http://localhost:3000`).
3. Navigate to a representative workflow run view.
4. Capture a screenshot with your OS screenshot tool.
5. Keep the image local or publish it in external release notes/wiki as needed.

If you later decide to version screenshots in Git, establish a dedicated policy first
(image format, max dimensions, compression, and review requirements).
