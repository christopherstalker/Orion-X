# Developer Notes

## Repository layout

- `.github/`: CI and collaboration templates
- `docs/`: product and operational documentation
- `examples/`: sample workflow specifications
- `scripts/`: helper scripts for local checks and automation

## Scripts

- `pnpm lint` / `pnpm test` / `pnpm build` (if present)
- `pnpm changeset`
- `pnpm version-packages`
- `pnpm release`

## Conventions

- Use Conventional Commits.
- Keep PRs focused and reviewable.
- Update docs when behavior or interfaces change.
